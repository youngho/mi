# BDS Go/No-Go 실험 체크리스트

BDS 소프트웨어 개발(A-1~) 착수 전, 하드웨어 전제가 성립하는지 검증합니다.

## 사전 준비

- [ ] RPLIDAR A3 또는 S3 + USB-UART 케이블
- [ ] 빔프로젝터 + 천 스크린 (실제 매장 규격)
- [ ] 레이저 커튼 오프셋 2~3cm 설치 완료
- [ ] 안전 장비 (고글, 방탄 천 등)
- [ ] `tools/bds-capture` 빌드 또는 Unity BDS 씬 준비

## 실험 절차

### 1. 베이스라인 (빈 스크린)

1. 라이다 전원 및 시리얼 연결 확인
2. 30초간 raw 스캔 녹화 (`LidarRecordingService` 또는 `bds-capture record`)
3. 정적 배경 포인트 분포·노이즈 레벨 기록

### 2. 간섭 물체

| 물체 | 기대 결과 | 실제 결과 | Pass |
|------|-----------|-----------|------|
| 손가락 천천히 통과 | 지속 포인트 (탄환 아님) | | |
| 막대 빠르게 휘두름 | 다프레임 지속 | | |
| BB탄 1발 | 1~2 스캔 transient | | |

### 3. BB탄 100발 시험

- 발사 거리: _____ m
- 센서 모델: A3 / S3
- 기록 파일: `logs/gonogo_YYYYMMDD.bin`

**측정 항목:**

| 지표 | 목표 | 측정값 |
|------|------|--------|
| 검출률 (100발 중 유효 transient) | ≥ 70% | |
| 이중 검출률 | ≤ 5% | |
| 평균 검출 지연 | ≤ 50ms | |
| 천 흔들림 false positive (100발 후 10초) | ≤ 3건 | |

### 4. Go/No-Go 판정

**Go (진행):** 검출률 ≥ 70% 이고 이중 검출률 ≤ 10%

**Conditional Go:** 검출률 50~70% → 필터·센서 위치 튜닝 1주 추가 후 재시험

**No-Go (중단):** 검출률 < 50%

No-Go 시 대안:

1. RPLIDAR S3 업그레이드
2. 듀얼 LiDAR (교차 검증)
3. IR/고속 카메라 하이브리드
4. 입력을 터치/마우스로 대체하고 PMS만 우선 개발

## 결과 기록

```
실험일:
실험자:
판정: Go / Conditional Go / No-Go
비고:
```

결과는 `logs/gonogo/`에 바이너리 로그와 함께 보관합니다.

# **🎯 \[기획서\] 고속 라이다(LiDAR) 연동 유니티 초고속 비비탄 직접 감지 시스템 개발**

## **1. 개요 (Overview)**

본 프로젝트는 빔프로젝터 스크린 환경에서 **실제 날아가는 직경 6mm의 BB탄 알갱이를 라이다(LiDAR) 레이저로 직접 낚아채어** 실시간 타겟 적중 좌표를 검출하는 시스템을 구축하는 것을 목표로 한다.

보급형 센서의 한계인 잔상 트릭을 배제하고, 초고속 샘플링(최소 16kHz \~ 32kHz 이상) 기반의 라이다 스펙을 활용한다. 미들웨어 없이 **유니티 C\#의 시리얼 통신(UART) 환경에서 고속 패킷을 직접 파싱**하며, 센서 체급이 올라가더라도 소프트웨어 아키텍처를 그대로 유지할 수 있도록 확장성 있는 파이프라인을 설계한다.

## **2. 하드웨어 구성 및 설치 레이아웃 (Hardware Layout)**

### **2.1 하드웨어 확장 스펙 가이드**

본 시스템은 초고속으로 이동하는 탄환을 직접 센싱해야 하므로 샘플링 레이트(Hz)가 핵심이다. 초기 빌드 및 상용화 단계에 맞춰 아래 센서 라인업을 호환하도록 설계한다.

  * **추천 센서 라인업:**
      * **RPLIDAR A3:** 샘플링 16,000 Hz / 측정 거리 25m (직접 감지 최소 미니멈 스펙)
      * **RPLIDAR S3:** 샘플링 32,000 Hz / 측정 거리 40m (상용화 및 높은 적중률 확보용 최고 스펙)
  * **통신 프로토콜:** UART (USB 가상 컴포트 연동), 고속 보레이트(BaudRate) 대응

### **2.2 설치 메커니즘 (레이저 커튼 셋팅)**

  * **위치 선정:** 라이다 센서를 천 스크린의 하단(또는 상단) 모서리에 배치한다.
  * **오프셋(Offset) 간격:** 라이다의 레이저 스캔 평면을 천 스크린 표면과 완전히 평행하게 맞추되, 스크린 앞쪽으로 **약 2\~3cm 공간을 띄워서** 레이저 평면(커튼)을 세운다.
  * **감지 원리:** 비비탄이 천에 부딪혀 발생하는 스크린의 울렁임이나 흔들림은 노이즈로 간주하여 필터링하고, 탄환이 천을 타격하기 직전 레이저 커튼을 통과하는 찰나의 순간(1ms 미만)을 고속 빔으로 직접 맞추어 좌표를 포착한다.

## **3. 유니티 C\# 데이터 파싱 아키텍처 (Data Parsing Architecture)**

초당 수만 번의 고속 거리 측정 패킷이 유니티의 메인 렌더링 프레임(FPS)을 저하시키지 않도록 **멀티쓰레딩(Multithreading) 및 비동기 큐(Queue)** 구조를 채택한다.

> **\[고속 라이다 센서\]** ──(UART 고속 스트리밍)──\> **\[유니티 백그라운드 쓰레드\]** (Raw 바이트 파싱)
> │
> (탄환 매칭 좌표만 Thread-Safe Queue에 삽입)
> ▼
> **\[유니티 게임 오브젝트\]** \<──(Update 메인 쓰레드)── **\[데큐(Dequeue) 수행 및 Raycast 타겟 판정\]**

### **3.1 C\# 고속 시리얼 수신 핵심 전략**

1.  **대용량 버퍼 할당:** 시리얼 포트 개방 시 ReadBufferSize를 충분히 크게 설정하여 고속 샘플링 시 데이터가 누락되는 오버플로우를 방지한다.
2.  **비동기 패킷 조립:** 백그라운드 쓰레드 내에서 하드웨어 고정 헤더 바이트를 상시 추적하고, 유효성 검사(Checksum/CRC)를 통과한 데이터만 순간적으로 계산 큐로 넘긴다.
3.  **데이터 정규화:** 센서 기종(A3, S3 등)에 따라 패킷 당 포함된 샘플 개수가 다르므로, 파싱 단에서 이를 **\[각도($\\theta$), 거리($r$)\]** 쌍으로 정규화하여 상위 모듈로 토스한다.

## **4. 소프트웨어 구현 주요 모듈 (C\# Scripts 구조)**

### **4.1 LidarHighSpeedReader.cs (고속 통신 및 파싱)**

  * System.Threading.Thread 기반으로 구동되며 하드웨어 패킷을 바이트 단위로 쪼개 각도 분해능과 거리 수치를 연산한다.
  * **기종 확장성:** 추후 센서 교체 시 패킷 구조체 프로토콜 매핑 함수만 교체할 수 있도록 인터페이스(ILidarParser) 구조를 적용한다.

### **4.2 LidarCoordinateMapper.cs (직교좌표 변환 및 스크린 매핑)**

  * 극좌표계($r, \\theta$) 데이터를 삼각함수를 통해 2D 공간의 직교좌표($X, Y$)로 1차 변환한다.
  * 라이다의 물리적인 설치 각도 왜곡을 보정하기 위해, 실제 프로젝트 빔 화면의 4개 모서리 지점을 타격하여 픽셀 좌표와 1:1 매칭시키는 **Homography 매트릭스 변환 알고리즘**을 포함한다.

### **4.3 LidarBulletFilter.cs (초고속 탄환 격발 트리거 필터)**

  * **공간 마스킹 (Static Masking):** 스크린 틀, 바닥, 천장 등 정적으로 항상 고정되어 잡히는 거리 데이터는 레이저 커튼 영역에서 미리 제외(Masking)한다.
  * **순간 점 트리거 (Instant Point Trigger):** 비비탄 알갱이는 속도가 매우 빠르기 때문에, 라이다 스캔 주기 기준 **오직 1\~2프레임 스캔 동안만 찰나에 나타났다가 다음 회전 때 완전히 사라지는 특징**을 가진다.
  * **판정 조건:** 정적 배경 영역 내에 없던 새로운 물체가 '단 1프레임 동안 특정 좌표에 출현 후 소멸'했을 때를 완벽한 비비탄 궤적으로 인지하고 메인 쓰레드에 격발 이벤트(OnBulletDetected(Vector2 screenPos))를 발생시킨다.

## **5. 단계별 개발 마일스톤 (Milestones)**

1.  **Phase 1: 고속 시리얼 데이터 스트리밍 안정화 (1주차)**
      * 16kHz 이상의 고속 패킷 환경에서 데이터 밀림이나 버퍼 오버플로우 없이 유니티 백그라운드 쓰레드가 상주하며 바이트를 정상적으로 분해하는지 검증.
2.  **Phase 2: Point Cloud 뷰어 빌드 (1주차)**
      * 라이다가 뿌려주는 각도/거리 데이터를 유니티 화면에 2D 점(Point)들로 실시간 시각화하여 센서 앞을 무언가 지나갈 때 점이 튀는 현상 확인.
3.  **Phase 3: 초고속 탄환 필터 알고리즘 구현 (2주차)**
      * 실제 비비탄을 발사하여 1프레임 미만으로 찍히는 점의 거리/강도(Confidence) 변화를 프로파일링하고, 팅겨 나간 후 천의 흔들림(지속 노이즈)을 지워버리는 컷오프(Cut-off) 필터 적용.
4.  **Phase 4: 인게임 4점 교정(Calibration) 시스템 구축 (2주차)**
      * 프로젝터 화면 모서리를 쏘아 라이다 왜곡 좌표를 게임 화면의 정규화 좌표($0.0 \~ 1.0$)로 매핑하는 UI 툴 완성.
5.  **Phase 5: 유니티 게임 콘텐츠 연동 (3주차)**
      * 최종 확정된 탄환 좌표를 유니티 2D/3D 공간 레이캐스트(Physics.Raycast)와 결합하여 타겟 붕괴, 스코어링 등의 게임 컨텐츠와 결합 및 최적화.
